Loops

Why Loops

Loops allow you do do the same piece of code multiple times. In some cases, you can just copy and paste the same piece of code multiple times, but what if you change your mind about something in the code and have to change it many times? Or what if you want to do the same thing 500 times? Loops make this much easier. In addition, sometimes you don't know how many times a piece of code needs to be executed. For example, if you wanted to print each item in a list, you would have to know how long the list is (you usually don't). Loops make dealing with lists of variable lengths much easier too.

while Loops

while loops execute a piece of code while a boolean condition is True. For example, we'll look at the following script:

x = 9
while x > 2:
    print(x)
    x = x - 2

This script prints x and subtracts 2 from x while x is greater than 2. It would output the following:

9
7
5
3

Next, we will trace exactly what happens in this code:

  1. First, x is set to 9
  2. Then we enter the loop
    1. On first test, 9 > 2 is True, so it prints 9 and subtracts 2 from x. x now contains 7
    2. On second test, 7 > 2 is True, so it prints 7 and subtracts 2 from x. x now contains 5
    3. On third test, 5 > 2 i True, so it prints 5 and subtracts 2 from x. x now contains 3
    4. On fourth test, 3 > 2 is True, so it prints 3 and subtracts 2 from x. x now contains 1
    5. On fifth, test, 1 > 2 is False, so it exits the loop
  4. That is the end of the script. (Note that x contains 1, not 3 - this confuses many people)

for Loops

Like while loops, for loops execute a piece of code multiple times. Instead of repeatedly testing a condition, for loops run for each element in a collection or iterator. We have not discussed iterators, but an iterator works essentially the same as a collection in a for loop except its values are calculated as needed rather than stored. The following shows an example for loop:

name_list = ["John", "Jim", "Bill", "Sarah"]
for single_name in name_list:
    print("Good morning, " + single_name)

This script would output the following:

Good morning, John
Good morning, Jim
Good morning, Bill
Good morning, Sarah

As you can see, the code is executed for every item in the list. the variable single_name is assigned to each item in the list on each consecutive iteration of the loop. We'll trace the program to see what exactly happens:

  1. First, the variable name_list is made into a list with 4 strings
  2. Then we enter the loop
    1. On the first iteration, single_name is assigned "John". It prints "Good morning, John".
    2. On the second iteration, single_name is assigned "Jim". It prints "Good morning, Jim".
    3. On the third iteration, single_name is assigned "Bill". It prints "Good morning, Bill".
    4. On the forth iteration, single_name is assigned "Sarah". It prints "Good morning, Sarah".
    5. That is the end of the list, so it exits the loop

Issues of for loops

For loops are extremely convenient in many cases, but they have limitations. For example, simple for loops often fail if:

  • The size of the loops changes
  • You want to change the values in the collection

The first issue is often solved by using a while loop. It is usually more applicable in cases where the size needs to change anyway. The second issue brings us to the next point - the range() function

range() Function

The range() is used for iterating over integer sequences. You can use it to produce sequences like {1, 2, 3, 4, 5} or {8, 6, 4, 2}. Note that in Python 3, range() produces a range object rather than a list, but you can cast it to a list with the list() builtin function. Here's an example of how it can be used:

for x in range(1, 5):
    print(x)

This would print the following:

1
2
3
4

The first parameter of range is the start number and the second is the end number. Note that it goes to the end number but does not include it. You can also provide a third parameter which specifies the increment, how much it increases by each time (default is 1). If that value is negative, it will go backwards. Both of these cases can be seen in the following:

for x in range(0, 10, 3):
    print(x)

This would print 0, 3, 6, and 9 (on newlines of course)

for x in range(10, 7 -1):
    print(x)

This would print 10, 9, and 8 (on newlines) because it goes up by -1 or down by 1

How range() is used for Mutating Lists

Because a simple for loop creates a copy of each item in the list, mutating is slightly different. Instead you would create a loop of indexes up to but not including the list length and change the list directly as seen in the following example:

list_ex = [4,5,5,9]
for index in range(0, len(list_ex)):
    list_ex[i] = 1

This loop replaces each item in the list with 1 by using an index. The variable list_ex would contain [1,1,1,1] after the loop.

break, continue, and pass Statements

Loops include several other keywords which make loops simpler to deal with. They are not necessary but they make your life easier.

  • break
    • The break keyword can be placed inside a loop so that when it is reached, the loop is automatically exited. It is often put inside a conditional so that it is only sometimes reached.
  • continue
    • Like the break keyword, continue exits the loop when it is reached. However, unlike break, it goes to the next iteration of the loop rather than exiting completely. In a while loop, the loop starts again by testing the conditional. In a for loop, it goes to the next item in the list and does the loop body again.
  • pass
    • Like in functions, pass just means that you intentionally have an empty loop. It can also be placed at the end of the loop body without harm but it does nothing. Like functions, you will rarely use the pass keyword.

Other useful looping functions

  • reversed() accepts a collection or iterable (meaning it can be used in a for loop) and returns the reverse of it. For example reversed([1,2,3]) would be 3, 2, 1. (Note that it is not a list by default but can be casted to one if necessary).
  • sorted() accepts a collection and returns it sorted by value (only works if the values are number-like)
  • zip() accepts any number of collections of the same length and allows you to iterate over all of them at once as in the following example:
    • x_list = [2,3,4]
y_list = [3,9,1]
z_list = [1,8,4]
for x, y, z in zip(x_list, y_list, z_list):
    print(x + y + z)
    • This would print 6, 20, and 9 (on newlines) because these are the sums of the first, second, and third items in each list
