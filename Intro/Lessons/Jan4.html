Why Programming

History of Programming

Programming is designed to make jobs easier. In the beginning, there was math, and it was hard. So man created computer, and it was good.
But the computer only understood binary machine code which looks like this:

0110100101010010101001010101010010101101010101010100101

So man created a language called assembly to make it easier to understand, and it looked like this:

msg: db "Hello World",10
len: equ $-msg
mov edx,len
mov ecx,msg
mov ebx,1
mov eax,4
int 0x80

This worked for a while, but man got greedy. From the greed (ambition) man created higher level programming languages which can be converted to machine code. There have been many variations, like C, Algol, but they still were not enough. Man wanted a language which looked and read like their own language, English. Thus, Python was created. In python, code is as readable as:

print("hello world")

What programming is not

Programming is not "smart." Through computer programming, the computer can do lots of stuff, but the computer can do nothing without a program to tell it what to do. The browser you are reading this with is simply a program. The operating system running it is a program. Even the thing before the operating system that says how to find the operating system is a program. You have to tell it what to do or it will not know how.

What programming is

Programming is the process of writing the code to make the computer do stuff. The computer cannot think on it's own so you have to say exactly what you want to do. The computer will not make assumptions.

Example: Making a peanut butter and jelly sandwich

Try to write instructions to tell someone really dumb how to make a peanut butter and jelly sandwich.

You most likely said something like "spread peanut butter on one piece of bread and spread jelly on another piece of bread, then put the two pieces together with the spreads between them." The person still has to make some assumptions. Instead, you would need to:

  1. Say exactly what you need to use
    • Two slices of bread
    • Peanut butter
    • Jelly
    • A knife
    • A spoon
  2. Describe every step necessary. You need to be simple and specific. For example:
    • Pick up bread and hold flat in your left hand
    • Transfer peanut butter to the face-up side of the bread
    • Spread peanut butter to be a flat layer covering all of face up surface
    • ...

Types of Programming Languages

Because there are so many programming languages, they are divided into many categories. Each is made to solve a specific issue (the specific issue may be to be general purpose). This section describes how Python is different from other programming languages.

Object-Oriented Programming (OOP) vs. Functional Programming

Object Oriented: In OOP, data is stored in collections called objects. Object represent various attributes of a single thing and how the thing can be changed. For example, an object is created to represent a person. It contains the person's characteristics (height, weight, hair color, etc.) as well as the things that person can do (i.e. walk, run, sleep). Java is a strong example of this type of language.

Functional: In functional programming, data is manipulated using math-like functions. The function is applied on variables and returns a variable. For example, the function multiply is applied on two variables x and y. If x=8 and y=4, then the function will return 32 becasue 8*4=32.

Python is multi-paradigm meaning it supports both types of programming. However, Python is primarily functional, so this course will not cover object oriented programming. This material is only covered in the advanced course.

Interpreted vs. Compiled Languages

In older languages like Java, code is compiled into machine code because the computer can only understand it that way. This means that a compiler program creates the machine code to complete the task written in the programming language.

Instead of this, Python is interpreted. It is converted to machine code when it is run. A program called an interpreter reads the programming language and completes the desired task as it is needed. This approach is slower than compiling, but it allows easier-to-write languages and provides a few added capabilities.

Static vs. Dynamic Typing

Because data can be of different types (integer, decimal number, character, string of characters), the computer must store different data differently. In statically-typed languages, you have to say what type of data each variable contains. On the contrary, Python is dynamically-typed. In such languages, you do not have to say what type of data a variable contains. This makes the language easier to write because you have one less thing to worry about. However, it is less safe because you will receive an error when you try to divide a number by a sentence1.

"Hello World" Example

Finally, we get to do real programming! The "Hello World" example is familiar to all programmers. In every language and every school, you almost always start by printing "Hello World." All it does it prints the words "Hello World."

You can find the program in the code bank here, but it is also here:

print("Hello World") # Prints, quite literally, "Hello World" to the screen.

  • print() is a function which accepts one input and uses the operating system to show it on the screen. The parameters are the things inside the parentheses which the function uses to do whatever it is meant to do.
  • The string ('string' just means a series of characters) "Hello World" is the information to be shown. It must be quotated because you want it printed exactly as it is written.
  • # signifies a comment. Anything you write after a # on a single line will be ignored. Writing comments to explain what you are doing is not just highly recommended but required in this class.
