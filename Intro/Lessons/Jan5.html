Example One - Basic Syntax

The first example will discuss the basic syntax of Python and the basics of variables. It can be found here, but here's an overview of the materials discussed:

  • Comments: Anything after a # is a comment and ignored by the interpreter.
  • Imports: You don't need this yet, but imports are used to use someone else's code
    • This import says to include the math library, which provides several math functions. The math package is included with Python by default.
    • Don't reinvent the wheel.
    • This program does not use the math library so it is unnecessary but included for example.
  • Variables and Assignment:
    • Variables store data (numbers, strings, etc.)
    • Variables are assigned with the assignment operator '='.
      • Assignment in form variable = literal
      • The literal data on the right side is stored into the variable on the left side.
    • Literals are data which are not variable (i.e. number 9 or string "hello")
    • Variables can change by reassignment
  • Displaying Variables - print() Function
    • print() is a function (this will be discussed in more detail soon) which displays data to the user
    • print() can be used to print a string literal as in print("hello") - note the quotation marks to signify literal
    • It can also be used print the value stored in a variable print(variable)
  • Importance of Tabs
    • Tabs are non-trivial in Python.
    • They signify that multiple lines of code go together.
    • For now, all of your code will need to be tabbed the same amount (probably not at all).
  • Basic Data Types
    • Integer: whole numbers (abbrev. "int") like 42
    • Floating-Point: decimal numbers (abbrev. "float") like 23.8
    • String: collection of characters like "example"
    • Boolean: True or False
    • There are other basic data types but they are not discussed now (or at all in the case of complex numbers)
Other notes about syntax:
  • Naming convention: Python is lenient on convention for naming variables but the most common is to use lower case words separated by underscores (i.e. this_is_a_variable)
    • Mixed case is also accepted which starts with a lowercase word and appends all following words with a capital first letter (i.e. thisIsAVariable)
  • You may have noticed this already, but capitalization matters. 'hi' is not the same as 'Hi'.
  • Comments are important. You may not see it now, but if you ever have to read someone else's code, you'll know. Start good habits now.

Basic Numeric Operations

Examples of the four basic math functions are shown in the second example. Here's an overview of the example:

  • Basic math functions (+, -, *, /) work the same in Python as in traditional math
  • When variables are used to calculate the value of other variables, their current values are used
  • Division is weird in other languages, but you don't have to worry about it in Python 3

Try It Out

Go ahead and play around with the Python interactive shell on the right. Put in some of the commands from the example script and make up some of your own. The best way to get familiar with Python is to play with it. If you're felling brave, you can write up some scripts and click the run button. Make sure to put print() statements if you want to see what's going on. Here are a some ideas:

  • Play around with parentheses and the four basic math functions in combination
  • Try putting math inside the print() function.

See what errors you can get too. You always want to know what not to do. Here are a few to start you off:

  • Literal or reserved word on the left of assignment operator '='
  • Using a variable that has not been given a value yet
  • Multiplying a string by a number (Especially try this one out, just to see one of the oddities of Python)
  • Adding strings (This one isn't necessarily an error but it doesn't do what you might expect) - this is called concatenation
