Using functions

Like in math, functions use variables to calculate values. For example, \( f(x) = 4x^2 \) is a function of x. For the comparable function 4 * x ** 2 in Python (or other programming languages), f is the function, x is a parameter and the value of f(x) is the return value. In a purely functional paradigm, functions act just as in math (they accept parameters and return a value based only on the parameters)1. For the example above, the function could be called by the following return_value = f(5). Note: "return" is a reserved word in Python. After the above statement, the variable return_variable would contain the return value of the function f with 5 as a parameter, or 100.

Static vs. Dynamic Typing and Type Safety

As discussed earlier, Python is a dynamically typed language. As such, variable types are not known until runtime, and types can change. Unlike in statically-typed languages, variables can be used as the wrong type and will not cause errors until they are used. Type safety (making sure you don't use the wrong type of object) is much more difficult in Python. The standard way is duck-typing which is essentially doing first and asking questions later (that is, tying potentially incorrect uses and dealing with errors if they occur). We will likely not discuss error handling in this lesson because it is not necessary to get started, but it is important if you plan to go farther.
